# 📚 מדריך שימוש - תרשים זרימת השרת

1. וודא שהשרת רץ על פורט 3001 (Frontend)
2. פתח דפדפן וגש ל: `http://localhost:3001/flow-chart.html`
3. תראה עמוד עם 4 טאבים בחלק העליון

---

### **📊 שלב 2: הבן את הטאבים**

#### **🌊 טאב "סקירה כללית"**
**מה תראה:**
- תרשים של כל השרת מלמעלה למטה
- איך בקשות מגיעות מהלקוח לשרת
- איך הנתונים זורמים בין MongoDB, Redis ובחזרה ללקוח

**איך להשתמש:**
1. עקוב אחרי החצים מלמעלה למטה
2. ראה איך בקשה עוברת: Client → Routes → Controller → Service → Database
3. שים לב לצבעים - כל צבע מייצג שכבה אחרת

#### **🛒 טאב "מערכת עגלה"**
**מה תראה:**
- איך הוספת מוצר לעגלה עובדת
- בדיקות ואימותים שקורים
- עדכון Redis ו-MongoDB

**איך להשתמש:**
1. עקוב אחרי השלבים: Add Request → Validation → Update → Response
2. ראה מה קורה אם המוצר לא קיים (חץ אדום לשגיאה)
3. הבן את ה-performance strategy למטה

#### **🛍️ טאב "מערכת מוצרים"**
**מה תראה:**
- איך קבלת רשימת מוצרים עובדת
- איך קבלת מוצר בודד עובדת
- מה קורה כשמוצר לא נמצא

**איך להשתמש:**
1. עקוב אחרי שני הזרמים: כל המוצרים VS מוצר בודד
2. ראה את מבנה הנתונים בתחתית
3. הבן מה קורה במקרה של שגיאה

#### **⚡ טאב "ביצועים"**
**מה תראה:**
- איך המערכת מטפלת במהירות
- מה קורה כשיש cache hit או miss
- איך המערכת מתמודדת עם שגיאות

**איך להשתמש:**
1. ראה את ההבדלים בזמני תגובה (1ms vs 200ms)
2. הבן את אסטרטגיית ה-caching
3. למד על fallback במקרה של בעיות
