#### 1. **PROJECT_BOOK.md** - ספר הפרויקט המלא

- **תיאור הפרויקט** - מה זה עושה ולמה

- **עיצוב מסד הנתונים** - אילו אוספים יש (Users, Products, Orders, Carts)

#### 2. **SERVER_QUICK_REFERENCE.md** - מדריך מהיר לשרת
זה המדריך שלך כדי להבין את הצד השרת שאתה אחראי עליו:
- **איך בקשה עובדת** - מה קורה מהרגע שהקליינט שולח בקשה
- **ארגון הקוד** - איפה כל דבר נמצא בקבצים
- **מושגים עיקריים** - JWT, password hashing, caching, stripe
- **איך להבין את הקוד** - צעד אחר צעד להבנה
- **משימות נפוצות** - איך להוסיף endpoint חדש, לשנות משהו, תיקון באגים
- **איפה הקבצים** - מדריך קבצים של השרת

#### 3. **FINAL_SUBMISSION_GUIDE.md** - הדרכה להגשה הסופית
כל מה שצריך לפני ההגשה:
- **רשימת בדיקה** - מה לבדוק לפני הגשה
- **מה להגיש** - קבצים ותיקייה בדיוק
- **איך להציג את הפרויקט** - מה לומר לפרופסור
- **תרגול הכנסה בחיים** - תסריט להדגמה חיה
- **שאלות וידובים** - שאלות שינשאלו וכמה לענות
- **רמות הצלחה** - מה צריך שתוכל לעשות

## מה הם שואלים בדרך כלל

### שאלה: איך עובדת ההרשמה?

**תשובה:**
"כשמשתמש מרשם, אנחנו:
1. משמרים את הסיסמה עם bcrypt (לא הסיסמה עצמה)
2. יוצרים משתמש בMongoDB
3. משדרים JWT token
4. הקליינט שומר את ה-token בpermission"

### שאלה: איך אתה מוודא שאפילו משתמש אחד לא יכול לראות הזמנות של אחר?

**תשובה:**
"כל endpoint מוגן בודק את ה-token. אנחנו מחלצים את user ID מהtoken ואז בודקים שהmeshimmer שייך לאותו משתמש:

```typescript
// לפני החזרת הזמנה
if (order.userId !== currentUserId) {
  throw Error('Unauthorized');
}
```"

### שאלה: למה Redis ולא רק MongoDB?

**תשובה:**
"MongoDB עובד טוב אבל חיפוש בעגלה מהר מאוד (המשתמש מוסיף ניתוח כל כמה שניות). Redis הוא בזיכרון ומהיר 50 פעמים יותר. אנחנו משמרים גם לMongoDB כדי שהנתונים לא אבדים."

### שאלה: איך Stripe משדרת תשלומים?

**תשובה:**
"המשתמש לא שולח את כרטיס האשראי לנו - זה נשלח ישירות ל-Stripe. אחרי תשלום מוצלח, Stripe שולחת לנו webhook. אנחנו מעדכנים את הסטטוס של ההזמנה. זה בטוח יותר."
